# Plans Archive Skill — Complete Reference

This skill manages Claude Code implementation plans across projects using a **global index** (`~/.claude/plans/INDEX.md`) paired with **project-local plan directories**.

## Core Structure

Plans live in two places:
- **`~/.claude/plans/<random-slug>.md`** — Auto-generated by Claude Code (read-only)
- **`<project>/plans/YYYY-MM-DD-plan-slug.md`** — Named copies with metadata headers

The global INDEX.md tracks all plans, mapping slugs to project locations and status.

## Key Workflows

**Archiving:** User says "archive this plan" → you locate content, confirm project/title, show preview, write files (with header block), update INDEX.md, confirm completion.

**Listing:** Read INDEX.md and display the full table; offer to load any plan.

**Retrieving:** Search INDEX.md, read matching local plan file, handle ambiguity by asking.

**Marking Executed:** Update status in INDEX.md, optionally add timestamp to plan header, suggest devlog entry.

**Session Start:** Check INDEX.md for active plan on current project; load silently or ask if uncertain.

## Local Filename Format

`YYYY-MM-DD-plan-slug.md` — archive date + 3–5 word lowercase-hyphenated slug from plan title.

## INDEX.md Table Columns

| Field | Notes |
|-------|-------|
| Date | Archive date |
| Slug | Random slug from `~/.claude/plans/` |
| Project | Short project name |
| Title | Plan H1 heading |
| Local Plan | Relative path to project plan file |
| Status | `active`, `archived`, `superseded`, `executed` |

Newest rows first. One slug can map to multiple projects.

## Proactive Trigger

At end of finalized plan-mode session, suggest: *"This plan looks finalized — want me to archive it to `<project>/plans/`?"* Skip for exploratory/draft/partial plans.

## Writing Plans: Header Block

```markdown
> Source: `~/.claude/plans/<slug>.md`
> Archived: YYYY-MM-DD · Project: [name]
> Maintained by [claude-plans-skill](https://github.com/code-katz/claude-plans-skill).

---

[plan content verbatim]
```

## Error Handling

Missing slug file → fall back to conversation extraction. Missing INDEX.md → create fresh. Ambiguous title/project → ask before writing. Unclear project directory → confirm with user.

## No Git Integration

Plans are archived locally only — no commits, no pushes, no repo requirements.
